# ReFlex Калькулятор ФЭМ

Интерактивный Streamlit-калькулятор для моделирования выхода на рынок ReFlex по моделям `Model A`, `Model B`, `Model A+B`.

## Актуальный scope

- Горизонт расчета: `1..36` месяцев (рыночная фаза)
- **R&D фаза до старта продаж** — отдельный период со своими постоянными расходами из инвестиционного банка
- Целевая безубыточность с автоподбором цены **по NPV** (дисконтированный CF)
- Ставка дисконтирования: задаётся как годовая %, пересчитывается в месячную по формуле `r_m = (1+r_год)^(1/12)−1`
- NPV-серия: кумулятивный дисконтированный CF, отображается на графике рядом с Cumulative CF
- 3 бизнес-модели + готовые сценарии `conservative/base/optimistic`
- Кастомные статьи затрат (fixed/variable)
- Переключаемый режим ввода параметров: `Слайдер` / `Ручной ввод`
- Автосохранение параметров при изменении настроек в sidebar
- Аудит формул (трейс + сверка с фактическими расчетами)
- Экспорт: Excel с формулами, Excel с данными, JSON, markdown snapshot для БП_04
- Слепок конфигурации: перенос всех настроек моделей между компьютерами (JSON import/export)
- График структуры revenue по выбранному месяцу + отдельный когортный график

## Логика моделей

- `Model A (B2B)` - клиника покупает парк + платит подписку/устройство/месяц
- `Model B (B2B2C)` - пациент платит аренду, клиника удерживает комиссию
- `Model A+B` - стартовый парк от клиники + арендная монетизация

Для `Model B` и `Model A+B` учитываются:
- когорты пациентов
- срок реабилитации (`rehab_duration_months`)
- освобождение устройств
- автоматическое расширение пула при дефиците

## NPV и ставка дисконтирования

В сайдбаре (блок **«📉 NPV (ставка дисконтирования)»**) задаётся годовая ставка дисконтирования:

- Месячная ставка: `r_m = (1 + r_год)^(1/12) − 1`
- NPV(T) = `Σ CF_t / (1 + r_m)^t` — кумулятивный дисконтированный CF
- При ставке **0%** NPV совпадает с Cumulative CF (обратная совместимость)

**Влияние на предиктор минимальной цены:**

Минимальная цена подписки/аренды теперь ищется из условия:

```
NPV(N)  ≥  Начальные_вложения + Запас × Σ Costs_m   (m = 1..N)
```

Это строже, чем старое условие по CumCF: при положительной ставке дисконтирования NPV < CumCF,
поэтому минимальная цена при высокой ставке будет выше, чем при нулевой.

Ориентир для параметра: **ключевая ставка ЦБ + премия за риск** (20–35% для российского стартапа).

## R&D фаза (до старта продаж)

### Как включить

В сайдбаре найдите блок **«🔬 R&D фаза»** и включите тумблер **«Включить R&D фазу перед продажами»**.

### Что настраивается

- **Длительность R&D** — от 1 до 24 месяцев (слайдер)
- **Категории расходов** — до 8 статей с редактируемыми названиями
- **Расходы по месяцам** — кнопка «📋 Заполнить расходы R&D по месяцам» открывает таблицу:
  - строки = категории, столбцы = R&D месяцы
  - каждая ячейка = расход в рублях для данной категории и месяца

### Ключевые правила

| Правило | Детали |
|---------|--------|
| Источник R&D расходов | Только банк инвестиций («Начальные вложения») |
| Превышение банка | Заблокировано: нельзя сохранить расходы R&D > банк |
| Нумерация на графиках 2–6 | Не изменяется: М1 = 1-й рыночный месяц |
| График 1 | Показывает R&D период (янтарный фон) + рыночный период |
| NPV | `CF_рыночный_t / (1+r_m)^(rnd_months + t)` — сдвиг от реального момента инвестиций |

### NPV с R&D поправкой

При включённой R&D фазе рыночный CF дисконтируется с учётом реального сдвига:

```
NPV(T) = -I₀ + Σ CF_t / (1 + r_m)^(rnd_months + t)
```

Это правильно: деньги от продаж получены позже момента инвестиций, поэтому ставка дисконтирования применяется за всё время с момента вложений.

### Конфигурация (слепок)

R&D настройки сохраняются в JSON-слепке конфигурации (`schema_version=5`). Старые слепки (v1–v4) при загрузке дают `rnd_enabled=False` — обратная совместимость гарантирована.

### Экспорт

- **Excel с данными / с формулами** — добавляется лист «R&D расходы»
- **Снимок для БП_04** — добавляется раздел «R&D фаза»
- **БП МСП** — в БДДС добавляются R&D месяцы до рыночных; в Инвест DCF — с поправкой на месяц offset

## Важные ограничения

- Mini what-if по допущениям A01-A04 — аналитический режим интерпретации, не замена полевой валидации гипотез
- Финальные pricing/GTM решения требуют валидации A01-A04 из бизнес-плана (используйте sensitivity и диапазоны до закрытия гипотез)
- NPV-предиктор — сценарный инструмент: задаёт нижнюю границу цены при заданной ставке дисконтирования. Ставка выбирается вручную исходя из cost of capital проекта.

## Перенос настроек между устройствами

- В sidebar откройте блок `Слепок конфигурации`.
- Нажмите `Скачать слепок конфигурации` и передайте JSON-файл коллеге.
- На другом компьютере загрузите файл через `Загрузить слепок конфигурации`.
- Импорт покажет preflight:
  - отсутствующие в файле новые поля будут заполнены `0`/дефолтом,
  - удаленные в текущей версии поля будут показаны как предупреждение с выбором `Игнорировать и применить` или `Отменить импорт`.
